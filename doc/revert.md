# git revert HEAD^

* 意图仅仅是撤消上上次操作，上上次以后的操作都不会撤消
* 根据上上次的提交记录，尝试生成一个反向的提交结果。

```
  p1
```

```
  p1
  p2
```

```
  p1
  p2
  p3
```

```
  p1
  p4
  p3
```

  **git revert HEAD^**

  理想状态是文件从

```
  p1
  p2
  p3
```

  回滚到

```
  p1
  p2
```
  如果确实是这种操作就不会有冲突

  但现在实际上是文件从

```
  p1
  p4
  p3
```

  回滚到
```
  p1
  p2
```
  那么就要用户自己解决冲突
